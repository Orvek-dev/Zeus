# Security Policy

Zeus is governed and local-first by default.

## Defaults

- No remote telemetry is enabled by the package.
- Local runtime state should stay outside source control.
- Provider credentials should be referenced by environment variable or external secret manager name, never copied into configs, tests, logs, or evidence.
- Secret-looking values are redacted in runtime surfaces that handle credential-like text.
- Tool, connector, transport, gateway, and live provider paths should pass through authority, lease, and evidence boundaries before execution.
- Dry-run and live-capable paths should remain distinguishable in code and review.

Provider Runtime status commands do not open network access or read credential material. Provider Runtime local deterministic smoke does not open network access or read credential material. Provider Runtime external receipt smoke requires a scoped environment secret reference and binds remote policy, preflight, receipt, audit, and redaction before reporting readiness, but it still does not claim unrestricted or production provider execution. MCP Runtime status, setup, list, inspect, and login dry-run commands do not open network access, start subprocesses, or read credential material. MCP Runtime governed test uses a fake client and cleanup receipt, not a production remote MCP server. Platform Runtime status and API dry-run commands do not bind sockets, start hosted servers, or execute live handlers. Platform Runtime gateway/session smoke uses temporary local stores and cleanup receipts. MCP resources/prompts remain disabled by default, production remote MCP server execution remains blocked, external gateway delivery remains blocked, webhook production execution remains disabled, browser live navigation remains blocked, Docker/SSH/remote sandbox execution remains blocked, cross-session memory search remains denied by default, and network egress remains denied by default outside explicitly leased provider/MCP/gateway receipt scenarios. This release does not claim production external provider execution, remote MCP production execution, hosted API daemon readiness, external gateway/webhook production execution, browser live execution, remote sandbox execution, unrestricted production live readiness, unattended execution, self-modifying workflows, automatic memory writes from ordinary agent turns, ontology promotion, learned-rule promotion, active rule writes, or hard-isolated remote execution by itself.

Before production use, live MCP tools, external AI APIs, browser/terminal automation, cron workers, remote sandboxes, networked gateways, and memory/ontology promotion paths should be wired through explicit authority grants, runtime leases, credential-scope binding, retention policy, human approval when needed, sandbox egress policy, audit evidence, and rollback behavior.
